## 🔐 Формат файлов

### Расширение: `.bgz`

**Анализ заголовка:**
```
00000000: 0000 0000 0000 0000 0000 0000 0000 0000  ................
00000010: 944f c2c4 32eb be38 a4a2 ac3c 4bcf 9bcb  .O..2..8...<K...
```

**Характеристики:**
- Первые 16 байт: нули (возможно, заголовок)
- Байты 16-31: похоже на хеш или подпись
- Остальное: сжатые/зашифрованные данные

**Не является:**
- ❌ Стандартным gzip (не распаковывается)
- ❌ ZIP архивом
- ❌ TAR архивом
- ❌ UBI образом

## 🔍 Попытки анализа

### Метод 1: Binwalk

```bash
binwalk k0u_700G_update.bgz
```

**Результат:** ❌ Ничего не найдено

---

### Метод 2: Gunzip

```bash
gunzip -c k0u_700G_update.bgz > output.bin
```

**Результат:** ❌ Ошибка формата

---

### Метод 3: Анализ строк

```bash
strings k0u_700G_update.bgz | head -20
```

**Результат:** ❌ Только бинарные данные

---

### Метод 4: Hex анализ

```bash
xxd k0u_700G_update.bgz | head -50
```

**Результат:** ⚠️ Данные выглядят как шифрованный поток

## 🛠️ Что нужно для вскрытия

### Требуется:

1. **Дамп NAND** с рабочего устройства
2. **Анализ загрузчика** (U-Boot)
3. **Поиск ключей дешифровки** в прошивке
4. **Реверс-инжиниринг** утилиты обновления

### Инструменты:

- IDA Pro / Ghidra
- Logic analyzer (для сниффинга UART)
- NAND дампер
- Python скрипты для анализа

## 🔓 Потенциальные векторы атаки

### 1. UART доступ

**Где искать:** Тестовые площадки на плате

**Что нужно:**
- USB-UART адаптер (3.3V)
- Подключение к TX/RX/GND
- Скорость: 115200 или 921600

**Цель:** Получить доступ к консоли загрузчика

---

### 2. SSH/Telnet

**Порты для проверки:**
```bash
nmap -p 22,23,2222,8080 <IP_KS0>
```

**Возможные учетки:**
- root / root
- admin / admin
- icriver / icriver

---

### 3. Веб-интерфейс

**URL:** `http://<IP_KS0>/`

**Уязвимости:**
- CGI скрипты
- File upload
- Command injection

---

### 4. Сетевые сервисы

- 4999 (майнинг)

### Приоритет 2: Дамп прошивки

```bash
# Если есть SSH:
ssh root@<IP_KS0> "cat /dev/mtd0" > mtd0_dump.bin
ssh root@<IP_KS0> "cat /dev/mtd1" > mtd1_dump.bin
```
