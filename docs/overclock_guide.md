# ⚡ Гид по разгону Avalon Nano 3

**Версия:** 1.0  
**Дата:** 11 марта 2026

---

## 📋 Введение

Avalon Nano 3 поддерживает разгон через CGMiner API. Этот документ описывает безопасные методы разгона и предостережения.

---

## ⚠️ Предупреждения

**РАЗГОН ДЕЛАЕТСЯ НА ВАШ СТРАХ И РИСК!**

- ⚠️ Разгон может привести к перегреву
- ⚠️ Превышение напряжения может повредить чипы
- ⚠️ Нестабильный разгон вызывает ошибки HW
- ⚠️ Гарантия может быть аннулирована

**Рекомендации:**
- Начинайте с малых значений
- Тестируйте стабильность 24+ часа
- Следите за температурой (макс. 85°C)
- Имейте резервную копию конфигурации

---

## 🎛️ Параметры разгона

### Частота (Frequency)

| Параметр | Значение |
|----------|----------|
| Минимум | 25 MHz |
| Максимум | 800 MHz |
| Сток | 500-700 MHz |
| Шаг | 25 MHz |

**Влияние:**
- ✅ Выше частота = выше хешрейт
- ❌ Выше частота = выше потребление и нагрев
- ❌ Слишком высокая = ошибки HW

---

### Напряжение (Voltage)

| Параметр | Значение |
|----------|----------|
| Минимум | 1150 mV |
| Максимум | 1450 mV |
| Сток | 1200-1300 mV |
| Шаг | 10 mV |

**Влияние:**
- ✅ Выше напряжение = стабильнее на высоких частотах
- ❌ Выше напряжение = значительно выше нагрев
- ❌ Слишком высокое = повреждение чипов

---

### Work Level

| Уровень | Значение | Описание |
|---------|----------|----------|
| 0 | Low | Обогреватель (мин. потребление) |
| 1 | Medium | Средний режим |
| 2 | High | Майнинг (макс. производительность) |

## 🔧 Применение разгона

### Через API (Python)

```python
import socket
import json

def send_privileged(host, port, command, arg):
    """Отправка привилегированной команды"""
    sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    sock.settimeout(5)
    sock.connect((host, port))
    
    cmd_data = {"command": "privileged", "arg": f"{command}={arg}"}
    sock.sendall(json.dumps(cmd_data).encode() + b"\n")
    
    response = b""
    while True:
        chunk = sock.recv(4096)
        if not chunk:
            break
        response += chunk
        if len(chunk) < 4096:
            break
    
    sock.close()
    return response.decode('utf-8', errors='ignore')

# Применение разгона ко всем чипам
MINER_IP = "192.168.31.133"
API_PORT = 4028

# Разгон: 750 MHz, 1350 mV для всех 10 чипов
for chip_id in range(10):
    result = send_privileged(MINER_IP, API_PORT, "ascset", f"{chip_id},750,1350")
    print(f"Chip {chip_id}: {result}")

# Установка Work Level
result = send_privileged(MINER_IP, API_PORT, "setconfig", "work_level,2")
print(f"Work Level: {result}")
```

### Через bash (nc)

```bash
#!/bin/bash
MINER_IP="192.168.31.133"
API_PORT="4028"

# Функция отправки команды
send_cmd() {
    echo "{\"command\":\"privileged\",\"arg\":\"$1\"}" | \
    nc -w 3 $MINER_IP $API_PORT | strings
}

# Разгон всех чипов: 750 MHz, 1350 mV
for i in {0..9}; do
    echo "Chip $i: $(send_cmd "ascset=$i,750,1350")"
done

# Установка Work Level
echo "Work Level: $(send_cmd "setconfig=work_level,2")"

# Сохранение конфигурации
echo "Save: $(send_cmd "save")"
```

## 📈 Пошаговый процесс разгона

### Шаг 1: Базовая линия

Запишите сток значения:
```bash
echo '{"command":"config"}' | nc -w 3 192.168.31.133 4028 | strings
echo '{"command":"summary"}' | nc -w 3 192.168.31.133 4028 | strings
```

**Запишите:**
- Базовый хешрейт
- Базовую температуру
- Базовое потребление

---

### Шаг 2: Увеличение частоты

Увеличивайте частоту на 25-50 MHz:
```bash
# Увеличить на 50 MHz
for i in {0..9}; do
    echo "{\"command\":\"privileged\",\"arg\":\"ascset=$i,750,1300\"}" | \
    nc -w 3 192.168.31.133 4028 | strings
done
```

**Тестируйте 1-2 часа:**
- Следите за ошибками HW
- Проверяйте хешрейт
- Проверяйте температуру

---

### Шаг 3: Настройка напряжения

Если появляются ошибки HW, увеличьте напряжение:
```bash
# Увеличить напряжение на 50 mV
for i in {0..9}; do
    echo "{\"command\":\"privileged\",\"arg\":\"ascset=$i,750,1350\"}" | \
    nc -w 3 192.168.31.133 4028 | strings
done
```

**Тестируйте 2-4 часа:**
- Проверяйте стабильность
- Следите за температурой

---

### Шаг 4: Финальное тестирование

Запустите на 24+ часа:
```bash
# Проверка статистики каждые 5 минут
watch -n 300 'echo "{\"command\":\"stats\"}" | nc -w 3 192.168.31.133 4028 | strings'
```

**Критерии стабильности:**
- ❌ Ошибки HW < 0.1% от принятых
- ❌ Температура < 80°C
- ✅ Хешрейт стабильный

---

### Шаг 5: Сохранение

```bash
# Сохранить конфигурацию
echo '{"command":"privileged","arg":"save"}' | \
nc -w 3 192.168.31.133 4028 | strings
```

## 🐛 Решение проблем

### Ошибки HW растут

**Причина:** Слишком высокая частота или низкое напряжение

**Решение:**
1. Уменьшите частоту на 25-50 MHz
2. Увеличьте напряжение на 20-50 mV
3. Проверьте охлаждение

---

### Температура слишком высокая

**Причина:** Недостаточное охлаждение или высокое напряжение

**Решение:**
1. Уменьшите напряжение на 50-100 mV
2. Уменьшите частоту на 50-100 MHz
3. Улучшите охлаждение (вентиляторы, термопаста)

---

### Майнер не запускается

**Причина:** Экстремальные настройки

**Решение:**
1. Сбросьте настройки (перезагрузка)
2. Начните со сток значений
3. Повышайте постепенно

---

### Шары отклоняются

**Причина:** Нестабильный разгон

**Решение:**
1. Проверьте ошибки HW
2. Снизьте частоту
3. Увеличьте напряжение

## 📊 Мониторинг

### Скрипт мониторинга (Python)

```python
#!/usr/bin/env python3
import socket
import json
import time

def get_stats(host, port):
    sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    sock.settimeout(5)
    sock.connect((host, port))
    sock.sendall(b'{"command":"devs"}\n')
    response = sock.recv(4096).decode('utf-8', errors='ignore')
    sock.close()
    return response

MINER_IP = "192.168.31.133"
API_PORT = 4028

print("Мониторинг Avalon Nano 3 (Ctrl+C для выхода)")
print("=" * 60)

try:
    while True:
        stats = get_stats(MINER_IP, API_PORT)
        print(f"\n{time.strftime('%Y-%m-%d %H:%M:%S')}")
        print(stats[:500])  # Первые 500 символов
        time.sleep(60)
except KeyboardInterrupt:
    print("\nМониторинг остановлен")
```
