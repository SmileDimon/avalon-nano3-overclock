## ❌ Проблемы и решения

### 1. Майнер не отвечает

**Симптомы:**
- ❌ Нет ответа на ping
- ❌ Веб-интерфейс не грузится
- ❌ API не подключается

**Причины:**
- Проблемы с питанием
- Сетевые настройки
- Зависание майнера

**Решение:**
```bash
# 1. Проверьте питание
# - Убедитесь что блок питания подключен
# - Проверьте индикаторы на майнере

# 2. Проверьте сеть
ping 192.168.31.133

# 3. Перезагрузите майнер
# - Отключите питание на 10 секунд
# - Включите снова

# 4. Проверьте IP
# - Используйте IP сканер
# - Проверьте DHCP сервер
```

---

### 2. API не отвечает

**Симптомы:**
- ✅ Ping работает
- ✅ Веб-интерфейс работает
- ❌ API порт 4028 не отвечает

**Причины:**
- API отключен в настройках
- Брандмауэр блокирует
- CGMiner завис

**Решение:**
```bash
# 1. Проверьте порт
nc -zv 192.168.31.133 4028

# 2. Проверьте настройки API
curl http://192.168.31.133/cgconf.cgi

# 3. Включите API в настройках
# - Веб-интерфейс → Settings → API
# - Убедитесь что API enabled

# 4. Перезапустите CGMiner
echo '{"command":"privileged","arg":"restart"}' | nc -w 3 192.168.31.133 4028
```

---

### 3. Высокие ошибки HW

**Симптомы:**
- ⚠️ Hardware Errors > 1% от Accepted
- ⚠️Rejected шары растут

**Причины:**
- Слишком высокая частота
- Недостаточное напряжение
- Перегрев
- Нестабильный разгон

**Решение:**
```bash
# 1. Проверьте температуру
echo '{"command":"devs"}' | nc -w 3 192.168.31.133 4028 | strings

# 2. Снизьте частоту на 50-100 MHz
for i in {0..9}; do
    echo "{\"command\":\"privileged\",\"arg\":\"ascset=$i,650,1350\"}" | \
    nc -w 3 192.168.31.133 4028 | strings
done

# 3. Увеличьте напряжение на 50 mV
for i in {0..9}; do
    echo "{\"command\":\"privileged\",\"arg\":\"ascset=$i,700,1400\"}" | \
    nc -w 3 192.168.31.133 4028 | strings
done

# 4. Проверьте охлаждение
# - Очистите радиаторы
# - Проверьте вентиляторы
# - Замените термопасту
```

---

### 4. Перегрев

**Симптомы:**
- ⚠️ Температура > 80°C
- ⚠️ Троттлинг (снижение частоты)
- ❌ Отключение майнера

**Причины:**
- Недостаточное охлаждение
- Высокое напряжение
- Высокая температура окружающей среды

**Решение:**
```bash
# 1. Снизьте напряжение
for i in {0..9}; do
    echo "{\"command\":\"privileged\",\"arg\":\"ascset=$i,600,1250\"}" | \
    nc -w 3 192.168.31.133 4028 | strings
done

# 2. Уменьшите частоту
for i in {0..9}; do
    echo "{\"command\":\"privileged\",\"arg\":\"ascset=$i,500,1200\"}" | \
    nc -w 3 192.168.31.133 4028 | strings
done

# 3. Установите work_level = 1 (Medium)
echo '{"command":"privileged","arg":"setconfig=work_level,1"}' | \
nc -w 3 192.168.31.133 4028 | strings

# 4. Улучшите охлаждение
# - Добавьте вентиляторы
# - Улучшите вентиляцию помещения
# - Очистите пыль
```

---

### 5. Пул отключается

**Симптомы:**
- ⚠️ Pool status: Dead
- ⚠️ No stratum
- ❌ Get Failures растут

**Причины:**
- Проблемы с интернетом
- Пул недоступен
- Неправильные настройки пула

**Решение:**
```bash
# 1. Проверьте подключение к интернету
ping 8.8.8.8

# 2. Проверьте настройки пула
curl http://192.168.31.133/cgconf.cgi

# 3. Добавьте пул заново
echo '{"command":"privileged","arg":"addpool=stratum+tcp://pool.com:3333,user,pass"}' | \
nc -w 3 192.168.31.133 4028 | strings

# 4. Переключитесь на резервный пул
echo '{"command":"privileged","arg":"switchpool=1"}' | \
nc -w 3 192.168.31.133 4028 | strings

# 5. Проверьте DNS
nslookup pool.example.com
```

### 6. Низкий хешрейт

**Симптомы:**
- ⚠️ Хешрейт ниже ожидаемого
- ⚠️ MHS av < 3500 GH/s

**Причины:**
- Низкая частота
- Work Level = 0 или 1
- Чипы в ошибке

**Решение:**
```bash
# 1. Проверьте текущую конфигурацию
echo '{"command":"config"}' | nc -w 3 192.168.31.133 4028 | strings

# 2. Установите Work Level = 2
echo '{"command":"privileged","arg":"setconfig=work_level,2"}' | \
nc -w 3 192.168.31.133 4028 | strings

# 3. Увеличьте частоту
for i in {0..9}; do
    echo "{\"command\":\"privileged\",\"arg\":\"ascset=$i,700,1300\"}" | \
    nc -w 3 192.168.31.133 4028 | strings
done

# 4. Проверьте статус чипов
echo '{"command":"devs"}' | nc -w 3 192.168.31.133 4028 | strings

# 5. Перезапустите майнер
echo '{"command":"privileged","arg":"restart"}' | \
nc -w 3 192.168.31.133 4028 | strings
```

### 7. Чип не работает

**Симптомы:**
- ❌ DEV status: Dead
- ❌ Температура чипа = 0
- ❌ Хешрейт чипа = 0

**Причины:**
- Чип отключен
- Проблема с питанием
- Физическое повреждение

**Решение:**
```bash
# 1. Проверьте статус всех чипов
echo '{"command":"devs"}' | nc -w 3 192.168.31.133 4028 | strings

# 2. Попробуйте включить чип
echo '{"command":"privileged","arg":"ascset=5,700,1300"}' | \
nc -w 3 192.168.31.133 4028 | strings

# 3. Перезапустите CGMiner
echo '{"command":"privileged","arg":"restart"}' | \
nc -w 3 192.168.31.133 4028 | strings

# 4. Если не помогло - аппаратная проблема
# - Проверьте подключение питания
# - Проверьте контакты
# - Обратитесь в сервис
```

### 8. Веб-интерфейс не работает

**Симптомы:**
- ✅ Ping работает
- ✅ API работает
- ❌ Веб-интерфейс не грузится

**Причины:**
- Веб-сервер завис
- Порт 80 занят
- Проблемы с CGI

**Решение:**
```bash
# 1. Проверьте порт 80
nc -zv 192.168.31.133 80

# 2. Попробуйте HTTPS
curl https://192.168.31.133/

# 3. Перезапустите майнер
# - Отключите питание на 10 секунд
# - Включите снова

# 4. Используйте API вместо веб-интерфейса
echo '{"command":"summary"}' | nc -w 3 192.168.31.133 4028 | strings
```

## 📊 Мониторинг состояния

### Скрипт диагностики

```bash
#!/bin/bash
MINER_IP="192.168.31.133"

echo "=== Диагностика Avalon Nano 3 ==="
echo ""

# Ping
echo -n "Ping: "
if ping -c 1 -W 2 $MINER_IP > /dev/null 2>&1; then
    echo "✅ OK"
else
    echo "❌ FAIL"
    exit 1
fi

# API
echo -n "API (порт 4028): "
if nc -zv $MINER_IP 4028 2>/dev/null; then
    echo "✅ OK"
else
    echo "❌ FAIL"
fi

# Веб
echo -n "Веб (порт 80): "
if nc -zv $MINER_IP 80 2>/dev/null; then
    echo "✅ OK"
else
    echo "❌ FAIL"
fi

# Версия
echo ""
echo "=== Версия ==="
echo '{"command":"version"}' | nc -w 3 $MINER_IP 4028 | strings

# Сводка
echo ""
echo "=== Сводка ==="
echo '{"command":"summary"}' | nc -w 3 $MINER_IP 4028 | strings | \
grep -E "Elapsed|MHS|Accepted|Hardware"
```
